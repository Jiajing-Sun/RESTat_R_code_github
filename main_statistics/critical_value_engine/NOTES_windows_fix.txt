Windows-server fix
==================

What changed
------------
1. Automatic worker capping
   - The code no longer uses detectCores()-1 without checks.
   - It now computes a safe worker count using:
       * detected logical cores
       * current open R connections
       * the R connection limit
       * an OS safety cap (Windows default cap = 32 workers)

2. Clear parallel reporting
   - The run summary now reports:
       * workers requested
       * workers actually used
       * connection budget
       * OS safety cap
   - This avoids misleading messages on large Windows servers.

3. Chunked PSOCK execution
   - Replications are grouped into chunks before parLapplyLB.
   - This reduces scheduling overhead and is more stable on Windows.

How to override the worker count
--------------------------------
From the command line, for example:

  Rscript run_generate_streamingcurve_cv.R --ncores=16

If no --ncores is supplied, the code chooses a safe value automatically.
